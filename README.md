校园景点导航系统 — 作品描述
一、项目简介
本系统是一款基于 Java Swing 图形界面开发的校园景点导航系统，面向高校师生及来访人员，提供校园景点信息的可视化管理与景点间最短路径的智能查询服务。系统以山西大学坞城校区为实际应用场景，涵盖校园内 15 个主要景点（教学楼、图书馆、宿舍、食堂、校医院、体育馆、校门等），通过图数据结构建模校园道路网络，并运用 Dijkstra 最短路径算法实现任意两景点间的最优路线规划。

二、核心功能
功能模块	说明
显示景点	以列表形式展示所有景点的编号、名称、所属校区及详细介绍，支持浏览全校景点信息
添加景点	支持动态新增景点，输入编号、名称、校区和简介即可扩充景点数据库
删除景点	可按编号移除不再需要的景点记录，同步清理相关路径数据
显示路径	以邻接表形式展示所有景点间的连通关系与距离信息
修改路径	支持新增或调整景点间的路径距离，灵活适应校园道路变化
查询路径（核心）	输入起始与目标景点编号，系统自动计算并展示最短路径及途经节点、总距离
数据持久化	支持手动保存与退出时自动保存，确保数据不丢失
三、技术架构
开发语言：Java（JDK 8+）
图形界面：Swing（JFrame + CardLayout 多面板切换架构）
数据结构：邻接矩阵（Adjacency Matrix）建模校园无向带权图
核心算法：Dijkstra 单源最短路径算法
数据存储：文本文件（景点信息 Jingdian.txt、路径数据 ditu.txt）+ 对象序列化（source.cls）
项目结构：
school.functions — 业务逻辑层（景点管理、地图建模、路径计算）
school.jiemian — 界面表现层（主窗口、各功能面板）
data/ — 数据文件目录
res/ — 资源文件目录（校园地图图片）
四、系统设计亮点
图论建模：将校园抽象为无向带权图，景点为顶点、道路为边、距离为权重，通过邻接矩阵存储，双向连通确保路径查询的准确性。

Dijkstra 算法实现：完整实现了经典的最短路径算法，通过已确定集合 S 和待确定集合 U 的逐步扩展，保证找到全局最优解，并通过前驱回溯输出完整路径。

CardLayout 面板架构：主窗口采用卡片布局管理多个功能面板，实现界面的无缝切换，各功能模块解耦独立，便于维护和扩展。

数据双模式持久化：运行时通过序列化机制缓存数据对象，关闭时将修改写回文本文件，兼顾运行效率与数据安全。

面向对象设计：景点类 Jingdian 实现了 Comparable 和 Serializable 接口，支持排序比较与序列化存储；功能类统一实现 Function 接口，符合开闭原则。

五、应用场景
新生入学或来访人员快速了解校园布局
规划校内出行路线，节省步行时间
校园管理部门动态维护景点与道路信息
数据结构与算法课程的教学演示案例
